Conceptos Basicos

Computadores e Informatica

La informática es la ciencia que se encarga del estudio de la teoría, la aplicación y la tecnología de la información y los sistemas de procesamiento de datos. Los computadores son una de las herramientas más importantes que se utilizan en informática y son dispositivos electrónicos que pueden recibir, procesar y almacenar datos para realizar una amplia variedad de tareas. La relación entre computadores e informática es, por tanto, muy estrecha ya que los computadores son una herramienta fundamental en el campo de la informática y sin ellos, muchos de los avances tecnológicos y científicos que se han producido en este campo no habrían sido posibles.

La informática como disciplina científica y tecnológica nació en la segunda mitad del siglo XX, a partir de la necesidad de desarrollar herramientas y métodos para el procesamiento y almacenamiento de datos en grandes cantidades. Esto se debió en gran medida al desarrollo de los computadores, que permitieron realizar tareas complejas de manera rápida y precisa. Con el tiempo, la informática fue evolucionando y abarcando una amplia variedad de campos, desde la ingeniería hasta la psicología y la biología, convirtiéndose en una disciplina fundamental en la sociedad moderna.

Electronica y Hardware

La electrónica es una disciplina que se encarga del estudio y aplicación de la electricidad y el magnetismo. Los computadores son dispositivos electrónicos que utilizan componentes electrónicos, como circuitos integrados, transistores y diodos, para recibir, procesar y almacenar datos. Por tanto, la electrónica es una disciplina fundamental en el desarrollo y funcionamiento de los computadores. Sin la electrónica, los computadores no serían posibles ya que no habría manera de controlar y manipular la electricidad de manera precisa para realizar las tareas que se les asignan. Por lo tanto, existe una estrecha relación entre la electrónica y los computadores ya que la electrónica es la disciplina que permite el funcionamiento de estos dispositivos.

Un computador funciona utilizando un lenguaje de programación basado en un sistema de codificación binario, en el que se utilizan dos símbolos: 1 y 0. Estos símbolos se utilizan para representar y procesar datos en un computador. Por ejemplo, si queremos representar una l etra del alfabeto como "N" utilizando este sistema de codificación, podríamos utiliza r una secuencia de 8 dígitos binarios, como por ejemplo: 01001110. Esta secuencia de dígitos binarios se puede interpretar como un código que indica al computador qué letra del alfabeto es "N" como en norman. De esta manera, los computadores pueden procesar y almacenar datos utilizando secuencias de dígitos binarios que pueden ser interpretados y utilizados para realizar una amplia variedad de tareas.

Los circuitos son una serie de componentes electrónicos interconectados que trabajan juntos para realizar una tarea específica. Podríamos decir que un circuito es como una carretera en la que circulan diferentes tipos de vehículos (componentes electrónicos) que se desplazan por ella siguiendo un camino específico (circuito) para llegar a su destino (realizar una tarea específica). Por ejemplo, en un circuito de iluminación, los componentes electrónicos que se desplazan por la carretera (circuito) son los electrones que fluyen a través del conductor y la tarea que realizan es iluminar una bombilla. En un circuito de control de motores, los componentes electrónicos que se desplazan por la carretera (circuito) son los pulsos eléctricos que se envían a un motor y la tarea que realizan es controlar su velocidad y dirección de giro. De esta manera, podemos decir que los circuitos son un conjunto de componentes electrónicos interconectados que trabajan juntos para realizar una tarea específica, como si fueran diferentes tipos de vehículos que se desplazan por una carretera para llegar a su destino.

Lenguaje de Programacion

Un lenguaje de programación se comunica con un circuito a través de un conjunto de instrucciones que se envían al circuito para que éste las ejecute. Estas instrucciones se escriben en el lenguaje de programación y se traducen a un lenguaje que pueda ser entendido por el circuito. Por ejemplo, si queremos enviar una instrucción al circuito para que encienda una bombilla, podríamos escribir un programa en un lenguaje de programación como Python que contenga la instrucción "encender bombilla". Luego, este programa se traduce a un lenguaje que pueda ser entendido por el circuito, como por ejemplo lenguaje ensamblador o lenguaje máquina, y se envía al circuito. El circuito recibe esta instrucción y la ejecuta, en este caso encendiendo la bombilla. De esta manera, un lenguaje de programación se comunica con un circuito a través de un conjunto de instrucciones que se envían al circuito para que éste las ejecute y realice una tarea específica.

Los lenguajes de programación de alto nivel y de bajo nivel son dos tipos de lenguajes que se utilizan en informática para crear programas y aplicaciones. La principal diferencia entre ambos tipos de lenguajes es el grado de abstracción que ofrecen. Los lenguajes de programación de alto nivel son más abstractos y se parecen más al lenguaje humano, lo que los hace más fáciles de entender y utilizar para los programadores. Por otro lado, los lenguajes de programación de bajo nivel son más cercanos a la forma en que un computador entiende y procesa la información, lo que los hace más eficientes en términos de velocidad y rendimiento, pero también más difíciles de utilizar para los programadores. Algunos ejemplos de lenguajes de programación de alto nivel son Python, Java y C++, mientras que algunos ejemplos de lenguajes de programación de bajo nivel son Assembly

Redes

Una red computacional es un sistema de interconexión de dispositivos informáticos, que permite que estos dispositivos compartan información y recursos entre sí. Las redes computacionales pueden variar en tamaño y complejidad, desde una red local que conecta una pequeña cantidad de dispositivos en una sola ubicación, hasta redes globales como el internet, que conecta millones de dispositivos en todo el mundo.

Las redes internas de un comutador se refieren a la red de conexiones que existe dentro del comutador mismo. Un comutador es un dispositivo de red que se encarga de enrutar el tráfico de red entre diferentes dispositivos conectados a la red. Dentro del comutador, hay una red de conexiones que permite que los diferentes componentes del comutador se comuniquen entre sí y enruten el tráfico de red de manera eficiente.

En general, las redes computacionales se componen de una serie de componentes básicos, como dispositivos de red, cables de red y protocolos de red que se utilizan para establecer y mantener la conexión entre los dispositivos. También se utilizan aplicaciones y servicios de red para permitir que los dispositivos compartan información y recursos de manera eficiente.

Las redes en internet se utilizan para conectar millones de dispositivos en todo el mundo y permitir que estos dispositivos compartan información y recursos de manera eficiente. Para hacer esto, se utilizan una serie de protocolos que establecen las reglas y estándares que se deben seguir para que los dispositivos puedan comunicarse entre sí a través de la red.

Algunos de los protocolos más importantes utilizados en internet son el Protocolo de Internet (IP), el Protocolo de Transmisión de Hipertexto (HTTP) y el Protocolo de Transferencia de Correo Electrónico (SMTP). El Protocolo de Internet (IP) es el protocolo principal que se utiliza para enrutar el tráfico de red entre diferentes dispositivos en internet. El Protocolo de Transmisión de Hipertexto (HTTP) se utiliza para transferir información en forma de hipertexto entre dispositivos, lo que permite que se carguen páginas web en los navegadores. El Protocolo de Transferencia de Correo Electrónico (SMTP) se utiliza para transferir correos electrónicos entre dispositivos.

Además de estos protocolos básicos, existen muchos otros protocolos utilizados en internet para diferentes fines, como el Protocolo de Seguridad de Capa de Transporte (TLS), que se utiliza para cifrar la información que se transmite a través de la red, y el Protocolo de Descubrimiento de Servicio (SDP), que se utiliza para permitir que los dispositivos descubran y se conecten a servicios en la red. En general, los protocolos de red son esenciales para el funcionamiento de internet y la comunicación entre dispositivos en la red.

los medelos de arquitectura son principalmente punto a punto, estrella, red en malla, modelo de anillo, red, arbol, red en torre, bus etc..., en la mayor parte de los casos se combinan

El modelo cliente servidor es el mas ocupado en general en las empresas por lo que revisaremos este

El modelo cliente-servidor es un modelo de arquitectura de red en el que los clientes se comunican con un servidor central para acceder a recursos o servicios. En este modelo, el servidor es el encargado de proporcionar información o servicios a los clientes, que son los dispositivos que solicitan estos recursos.

Por ejemplo, cuando un usuario accede a un sitio web a través de su navegador web, el navegador actúa como cliente y se comunica con el servidor web que aloja el sitio para solicitar la página web. El servidor envía la página web al navegador, que la muestra al usuario.

modelo cliente-servidor tiene varias ventajas, como la capacidad de escalar para manejar una gran cantidad de solicitudes de clientes y la posibilidad de centralizar la gestión de recursos y servicios en el servidor. Sin embargo, también tiene algunas desventajas, como una mayor complejidad y una mayor dependencia del servidor central.

Arquitecrua de datos

La arquitectura de datos se refiere a la forma en que los datos se almacenan, se procesan y se utilizan en una empresa. Esta arquitectura incluye aspectos como la estructura de las bases de datos, la forma en que se integran los datos de diferentes fuentes y la implementación de políticas de seguridad de datos.

La arquitectura de datos tiene como objetivo facilitar el acceso a los datos de la empresa y garantizar su integridad, seguridad y calidad. Para lograr esto, la arquitectura de datos suele incluir elementos como:Bases de datos centralizada,Herramientas de integración de datos,Políticas de seguridad de datos,Herramientas de análisis de datos

Existen varios tipos de arquitecturas de datos, cada una con sus propias características y ventajas. Algunos ejemplos de arquitecturas de datos incluyen:

  • Arquitectura de base de datos centralizada: en esta arquitectura, todos los datos de la empresa se almacenan en una única base de datos centralizada. Esta arquitectura es adecuada para empresas pequeñas o con un volumen bajo de datos, pero puede no ser escalable para empresas más grandes.
  • Arquitectura de base de datos distribuida: en esta arquitectura, los datos de la empresa se almacenan en varias bases de datos que se encuentran en diferentes ubicaciones físicas. Esta arquitectura es más escalable que la arquitectura centralizada, pero puede ser más compleja de implementar y administrar.
  • Arquitectura de base de datos en nube: en esta arquitectura, los datos de la empresa se almacenan en una base de datos que se encuentra en una nube de computación. Esta arquitectura es muy escalable y permite acceder a los datos desde cualquier lugar, pero puede depender de la disponibilidad de internet.
  • Arquitectura de base de datos en memoria: en esta arquitectura, los datos de la empresa se almacenan en la memoria principal de los servidores en lugar de en un disco duro. Esta arquitectura ofrece un rendimiento muy alto, pero puede ser costosa y requerir una alta disponibilidad de energía.
  • Arquitectura de base de datos en tiempo real: en esta arquitectura, los datos se procesan y se entregan a los usuarios en tiempo real, lo que permite tomar decisiones en el momento.
  • Arquitectura de base de datos orientada a objetos: en esta arquitectura, los datos se almacenan y se procesan en forma de objetos, lo que permite una mayor flexibilidad y extensibilidad.
  • Arquitectura de base de datos multidimensional: en esta arquitectura, los datos se almacenan en forma de cubos multidimensionales, lo que facilita el análisis y la agregación de datos.
  • Arquitectura de base de datos en columna: en esta arquitectura, los datos se almacenan en columnas en lugar de filas, lo que puede mejorar el rendimiento en ciertas operaciones.

Cada tipo de arquitectura de datos tiene sus propias ventajas y desventajas, y la elección del modelo adecuado dependerá de las necesidades y características de la empresa en cuestión.

Rol de las bases de Datos

bases de datos son una parte fundamental de la arquitectura de datos, ya que son el lugar donde se almacenan y organizan los datos que son utilizados por una organización. En general, el rol de las bases de datos en la arquitectura de datos es proporcionar un lugar centralizado y seguro donde los datos pueden ser almacenados, recuperados y actualizados de manera eficiente y confiable. Esto permite a las organizaciones acceder a la información de manera rápida y sencilla, lo que a su vez les permite tomar decisiones informadas y mejorar sus operaciones.

En términos generales, las bases de datos pueden cumplir una variedad de funciones en la arquitectura de datos, como:

  • Almacenar grandes cantidades de datos de manera organizada y estructurada, de modo que puedan ser recuperados de manera rápida y eficiente.
  • Proporcionar un lugar centralizado donde los datos puedan ser compartidos y utilizados por diferentes aplicaciones y sistemas de una organización.
  • Permitir a las organizaciones realizar consultas complejas sobre sus datos, lo que les permite obtener información valiosa y tomar decisiones informadas.
  • Proteger la integridad y la seguridad de los datos, asegurando que solo personas autorizadas puedan acceder a ellos y evitando pérdidas o daños.

diferentes tipos de bases de datos, cada uno con sus propias características y ventajas. Algunos de los tipos más comunes de bases de datos son:

  • Bases de datos relacionales: estas bases de datos almacenan los datos en forma de tablas, donde cada fila representa un registro y cada columna representa un campo de datos. Las tablas están relacionadas entre sí mediante claves, lo que permite realizar consultas complejas sobre los datos.
  • Bases de datos no relacionales: estos tipos de bases de datos no utilizan un esquema de tablas relacionadas, sino que almacenan los datos de manera desestructurada. Esto las hace más flexibles y escalables que las bases de datos relacionales, pero también puede dificultar el acceso a los datos.
  • Bases de datos orientadas a objetos: estas bases de datos utilizan un modelo de datos basado en objetos, lo que les permite almacenar y recuperar datos en forma de objetos complejos y relacionados.
  • Bases de datos distribuidas: estos tipos de bases de datos se componen de varios servidores que trabajan juntos para almacenar y procesar los datos. Esto permite una mayor escalabilidad y disponibilidad, pero también puede ser más complejo de administrar.
  • Bases de datos en memoria: estas bases de datos almacenan los datos en la memoria principal del equipo en lugar de en disco, lo que les permite un acceso y un rendimiento mucho más rápidos que otros tipos de bases de datos.

Los gestores de bases de datos son programas que se encargan de administrar y gestionar las bases de datos, permitiendo almacenar, recuperar y actualizar los datos de manera eficiente y segura. Algunos de los gestores de bases de datos más utilizados son:

  • MySQL: es un gestor de bases de datos relacionales muy popular, utilizado en una amplia variedad de aplicaciones web y sistemas empresariales.
  • Microsoft SQL Server: es un gestor de bases de datos relacionales desarrollado por Microsoft, que ofrece un alto rendimiento y una amplia gama de características avanzadas.
  • Oracle Database: es uno de los gestores de bases de datos más completos y potentes del mercado, utilizado en entornos empresariales de alta exigencia.
  • MongoDB: es un gestor de bases de datos no relacionales, basado en documentos, que ofrece una alta escalabilidad y flexibilidad.
  • PostgreSQL: es un gestor de bases de datos relacionales libre y de código abierto, que ofrece un alto rendimiento y una amplia gama de características avanzadas.

Proceso de desarollo

áreas que intervienen en el desarrollo de software suelen ser:

  • Diseño: esta área se encarga de definir el aspecto y la funcionalidad del software, así como también la arquitectura y los componentes que lo conforman.
  • Desarrollo: esta área se encarga de implementar el código fuente del software, utilizando lenguajes de programación y herramientas de desarrollo.
  • Testing: esta área se encarga de probar el software para asegurarse de que funciona correctamente y cumple con los requisitos especificados.
  • Documentación: esta área se encarga de preparar la documentación técnica y de usuario del software, para que los usuarios puedan utilizarlo de manera adecuada.
  • Soporte: esta área se encarga de brindar asistencia a los usuarios y resolver problemas o incidencias que puedan surgir con el uso del software.

Metodologia General

Existen muchas metodologías diferentes para el desarrollo de software, cada una con sus propias ventajas y desventajas en función de las necesidades del proyecto. Algunas de las metodologías más comunes incluyen Agile, Scrum, Lean, y Waterfall. La metodología Agile se centra en la colaboración y la flexibilidad, y se divide en varios subenfoques como Scrum y Lean. La metodología Scrum utiliza equipos autónomos y iteraciones cortas para desarrollar software de forma rápida y eficiente. La metodología Lean se centra en la eliminación de desperdicios y la maximización del valor para el cliente. La metodología Waterfall se centra en el seguimiento de un proceso secuencial y estructurado para desarrollar software.

Pese a que en la mayoria de las empresas se intenta implementar una metodologia Agile, finalmente debido a la cultura termina transformandose en una metodologia hibrida, que genera deuda tecnica. Lo mas comun es que se vea una metodologia en cascada con trazas de Agile

La principal diferencia entre la metodología de cascada y las metodologías ágiles es que la primera es un enfoque secuencial y planificado, mientras que las segundas son enfoques iterativos y adaptativos. Las metodologías ágiles suelen ser más flexibles y eficientes que la metodología de cascada, pero requieren un mayor compromiso y trabajo en equipo.

Criptografia

La criptografía es la ciencia que se encarga del estudio de los métodos para proteger la información mediante el uso de códigos y algoritmos matemáticos. En otras palabras, la criptografía es la técnica utilizada para cifrar y descifrar mensajes para que solo puedan ser leídos por las personas autorizadas.

Algunos Metodos Criptograficos son:

  • Cifrado simétrico: Norman quiere enviar un mensaje confidencial a Camila. Ambos acuerdan utilizar la contraseña "norman1234" como clave secreta para cifrar y descifrar el mensaje. Norman utiliza esta contraseña para cifrar el mensaje y lo envía a Camila. Cuando Camila recibe el mensaje, lo descifra utilizando la misma contraseña.
  • Cifrado asimétrico: Norman quiere enviar un mensaje confidencial a una empresa. La empresa le proporciona su clave pública, que Norman puede utilizar para cifrar el mensaje. Norman utiliza esta clave para cifrar el mensaje y lo envía a la empresa. Cuando la empresa recibe el mensaje, lo descifra utilizando su clave privada, que solo es conocida por ella.
  • Firma digital: Camila quiere enviar un contrato electrónico a un cliente. Antes de enviar el contrato, utiliza su clave privada para generar una firma digital que se añade al documento. Cuando el cliente recibe el contrato, utiliza la clave pública de Camila para verificar que la firma digital proviene de ella y que el documento no ha sido modificado.
  • Hash: Norman quiere verificar la integridad de un archivo descargado de Internet. Primero, genera un hash del archivo original utilizando una función matemática. Luego, comprueba que el hash del archivo descargado coincide con el hash del archivo original. Si el hash es diferente, significa que el archivo ha sido modificado y no puede confiar en su integridad.

Seguridad en Redes

Ataque de denegación de servicio (DoS): Un ataque de DoS se produce cuando un atacante envía una cantidad masiva de tráfico a un servidor o red, con el objetivo de sobrecargarlos y hacerlos inaccesibles para los usuarios legítimos. Ataque de fuerza bruta: Un ataque de fuerza bruta consiste en intentar adivinar la contraseña de una cuenta o sistema mediante el uso de un programa que prueba diferentes combinaciones de caracteres hasta encontrar la correcta. Ataque de ingeniería social: Un ataque de ingeniería social es un intento de manipular a una persona para que revele información confidencial o realice una acción no deseada. Los atacantes pueden utilizar técnicas como el phishing o el spear phishing para llevar a cabo este tipo de ataque. Ataque de escaneo: Un ataque de escaneo consiste en explorar una red o sistema informático en busca de vulnerabilidades que puedan ser explotadas por el atacante. Los atacantes pueden utilizar herramientas de escaneo automatizadas para llevar a cabo este tipo de ataque.

Exploits

Un exploit es un programa o código diseñado para aprovechar una vulnerabilidad en un sistema o aplicación con el objetivo de obtener acceso no autorizado o causar daños. Los exploits suelen utilizarse en conjunción con otros tipos de ataques, como el phishing o el escaneo, para facilitar la ejecución de la acción maliciosa.

Otros Aspectos

Conectando Todo lo Aprendido

en un sistema de información, los diferentes componentes, como servidores, bases de datos y aplicaciones, se comunican entre sí mediante la transmisión de mensajes o paquetes de datos a través de la red. Estos mensajes pueden ser cifrados para proteger la privacidad de los datos y pueden ser enviados mediante diferentes protocolos de comunicación, como TCP/IP o HTTP.

Rol de las APIS

Las APIs (Application Programming Interfaces, o Interfaces de Programación de Aplicaciones en español) son conjuntos de protocolos y herramientas que permiten a diferentes aplicaciones comunicarse y compartir datos entre sí. Las APIs pueden ser de diferentes tipos, según su alcance y propósito. Algunos de los tipos de APIs más comunes son:

  • APIs de sistema: estas APIs permiten a las aplicaciones acceder a los recursos del sistema operativo, como por ejemplo la memoria, el disco duro o las entradas/salidas.
  • APIs de aplicación: estas APIs permiten a las aplicaciones interactuar con otras aplicaciones, compartiendo datos y funcionalidades entre ellas.
  • APIs de red: estas APIs permiten a las aplicaciones comunicarse a través de una red, enviando y recibiendo datos a través de diferentes protocolos.
  • APIs web: estas APIs permiten a las aplicaciones acceder a recursos de internet, como por ejemplo páginas web, imágenes o datos en formato JSON o XML.

RCP, REST, SOAP, JSON, REST API, las apis web mas conocidas

  • REST: estas APIs utilizan el protocolo HTTP y el estilo de arquitectura REST (Representational State Transfer) para permitir a las aplicaciones enviar y recibir datos en formato JSON o XML. Las APIs REST suelen ser muy flexibles y fáciles de implementar, y se utilizan en una amplia variedad de aplicaciones web.
  • APIs SOAP: estas APIs utilizan el protocolo HTTP y el estándar SOAP (Simple Object Access Protocol) para permitir a las aplicaciones intercambiar mensajes en formato XML de manera estructurada y segura. Las APIs SOAP suelen ser más complejas que las APIs REST, pero ofrecen un mayor nivel de integración y seguridad.
  • APIs GraphQL: estas APIs utilizan un lenguaje de consultas propio llamado GraphQL para permitir a las aplicaciones acceder a recursos de internet de manera eficiente y flexible. Las APIs GraphQL suelen ser más modernas que las APIs REST o SOAP, y permiten una mayor flexibilidad en la recuperación de datos.
  • APIs JSON-RPC: estas APIs utilizan el protocolo HTTP y el formato JSON-RPC (JSON Remote Procedure Call) para permitir a las aplicaciones invocar procedimientos remotos de manera sencilla y ligera. Las APIs JSON-RPC suelen ser muy simples y fáciles de implementar, y se utilizan en aplicaciones web de bajo nivel de complejidad.
